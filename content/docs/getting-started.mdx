title: "Getting started"
description: "Install the CLI, scaffold a service, and deploy to Selium."
---

This guide sets up a local runtime, builds a tiny service, and runs it via the remote client.

## Prerequisites

- Rust (stable)
- `rustup target add wasm32-unknown-unknown`

## 1. Install the runtime and CLI

If you use the install script, you get `selium-runtime` (host) and `selium` (remote client CLI):

```bash
curl --proto '=https' --tlsv1.2 -sSfL https://selium.com/install.sh | sh
```

Prefer building from source? From the Selium repos:

```bash
# Runtime
cargo build -p selium-runtime --release

# Remote client and optional modules
cargo build -p selium-remote-client-server --target wasm32-unknown-unknown --release
cargo build -p selium-switchboard-server --target wasm32-unknown-unknown --release
cargo build -p selium-atlas-server --target wasm32-unknown-unknown --release
```

## 2. Create a work directory

```bash
mkdir -p selium-work/{certs,modules}
```

The runtime expects:

- `selium-work/certs` for TLS material
- `selium-work/modules` for WASM modules (your services and system modules)

## 3. Create a service

`src/lib.rs`:

```rust
use selium_userland::entrypoint;
use tracing::info;

#[entrypoint]
fn hello() {
    info!("hello from selium");
}
```

`Cargo.toml`:

```toml
[package]
name = "hello"
version = "0.1.0"
edition = "2024"

[lib]
crate-type = ["cdylib"]

[dependencies]
selium-userland = "1.0.0-alpha.5"
tracing = "0.1"
```

Build to WASM and copy into the work dir:

```bash
cargo build --release --target wasm32-unknown-unknown
cp target/wasm32-unknown-unknown/release/hello.wasm selium-work/modules/
```

## 4. Generate certs and start the runtime

```bash
selium-runtime generate-certs --output-dir selium-work/certs

selium-runtime --work-dir selium-work \
  --module 'path=modules/selium_remote_client_server.wasm;capabilities=ChannelLifecycle,ChannelReader,ChannelWriter,ProcessLifecycle,NetQuicBind,NetQuicAccept,NetQuicRead,NetQuicWrite;args=utf8:localhost,u16:7000' \
  --module 'path=modules/selium_switchboard_server.wasm;capabilities=ChannelLifecycle,ChannelReader,ChannelWriter,SingletonRegistry' \
  --module 'path=modules/selium_atlas_server.wasm;capabilities=ChannelLifecycle,ChannelReader,ChannelWriter,SingletonRegistry'
```

If you do not need service discovery or messaging, you can omit `switchboard` and `atlas`.

## 5. Start your service with the CLI

```bash
selium --cert-dir selium-work/certs \
  start hello.wasm start \
  --capabilities ChannelLifecycle,ChannelReader,ChannelWriter \
  --attach
```

You will see `Started process (id=...)`. To stop it later:

```bash
selium --cert-dir selium-work/certs stop <id>
```
